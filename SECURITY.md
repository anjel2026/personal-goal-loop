# Security Policy

## Supported Versions

The project is pre-1.0. Security fixes target the latest `main` branch until versioned releases begin.

## Reporting

Do not open public issues with real personal data, business records, workbook samples, credentials, or identity maps.

Use a private report channel for sensitive cases. If no private channel exists yet, open a public issue that describes the class of problem without attaching data.

## Data Handling

This repository should only contain synthetic fixtures. Before release, scan the public tree with a private denylist outside the repository.
